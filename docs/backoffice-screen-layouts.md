## 혜택 카테고리 관리 화면 (Benefit Categories Management)

**경로**: `/benefits`
**파일**: `apps/pickly_admin/src/pages/benefits/BenefitCategoryList.tsx`

### 화면 구성

#### 1. 헤더 영역
```
┌─────────────────────────────────────────────────────────────┐
│  [H4] 혜택 카테고리                    [새 카테고리 버튼 +] │
└─────────────────────────────────────────────────────────────┘
```
- 좌측: 페이지 제목 "혜택 카테고리" (Typography variant="h4")
- 우측: "새 카테고리" 버튼 (Button variant="contained", startIcon: AddIcon)
  - 클릭 시: `/benefits/new` 페이지로 이동

#### 2. 데이터 그리드 (DataGrid)
```
┌──────┬───────────┬────────┬──────┬──────┬───────────┬──────┐
│ 이름 │ 슬러그    │ 설명   │ 순서 │ 상태 │ 등록일    │ 작업 │
├──────┼───────────┼────────┼──────┼──────┼───────────┼──────┤
│ 주거 │ housing   │ 주거...│  1   │ ✅   │2024-01-15 │ ✏️🗑️│
│ 교육 │ education │ 교육...│  2   │ ✅   │2024-01-15 │ ✏️🗑️│
│ 교통 │ transport │ 교통...│  3   │ ✅   │2024-01-15 │ ✏️🗑️│
└──────┴───────────┴────────┴──────┴──────┴───────────┴──────┘
```

**컬럼 구조**:
1. **이름** (name)
   - flex: 1, minWidth: 150
   - 표시: 카테고리 이름

2. **슬러그** (slug)
   - flex: 1, minWidth: 120
   - 표시: URL 친화적 식별자
   - 예: "housing", "education"

3. **설명** (description)
   - flex: 2, minWidth: 200
   - 표시: 카테고리 설명 (최대 100자, 초과 시 "...")

4. **순서** (display_order)
   - width: 100, align: center
   - 표시: 정렬 순서 번호

5. **상태** (is_active)
   - width: 100
   - 표시: Chip 컴포넌트
     - 활성: 🟢 "활성" (green)
     - 비활성: 🔴 "비활성" (red)

6. **등록일** (created_at)
   - width: 140
   - 형식: "yyyy-MM-dd HH:mm"
   - 예: "2024-01-15 14:30"

7. **작업** (actions)
   - width: 120, sortable: false
   - 버튼:
     - ✏️ 수정 버튼 (EditIcon) → `/benefits/{id}/edit`
     - 🗑️ 삭제 버튼 (DeleteIcon, color="error")
       - confirm 다이얼로그 표시
       - 확인 시 삭제 mutation 실행

**데이터그리드 설정**:
- 페이지 크기 옵션: 10, 25, 50, 100
- 기본 페이지 크기: 25
- 행 선택 비활성화
- 정렬 활성화 (등록일 기준 내림차순)

**필터 컴포넌트**:

  - 옵션: 전체, 모집중, 마감, 임시저장, 예정
  - size: "small"

```
┌──────────────┬──────┬──────────┬────────┬──────────┬──────────┬──────┬─────────┬──────┐

│ (부제목)     │      │          │        │          │          │      │         │      │
└──────────────┴──────┴──────────┴────────┴──────────┴──────────┴──────┴─────────┴──────┘
```

**컬럼 구조**:

1. **제목** (title)
   - flex: 2, minWidth: 200
   - 표시:
     - 상단: 제목 (fontWeight: 500)
     - 하단: 부제목 (caption, color: text.secondary)
   - 렌더링: Box + Typography 조합

2. **기관** (organization)
   - flex: 1, minWidth: 150
   - 표시: 주관 기관명

3. **상태** (status)
   - width: 120
   - 표시: Chip 컴포넌트
     - 🟢 모집중 (green, "recruiting")
     - 🔴 마감 (red, "closed")
     - ⚫ 임시저장 (default, "draft")
     - 🔵 예정 (blue, "upcoming")

4. **카테고리 ID** (category_id)
   - width: 120
   - 표시: UUID의 첫 8자 + "..."
   - 예: "abc12345..."

5. **신청 시작일** (application_period_start)
   - width: 140
   - 형식: "yyyy-MM-dd"
   - 없으면: "-"

6. **신청 마감일** (application_period_end)
   - width: 140
   - 형식: "yyyy-MM-dd"
   - 없으면: "-"

7. **조회수** (views_count)
   - width: 100, align: center
   - 형식: 천 단위 콤마 (예: "1,234")
   - 기본값: 0

8. **등록일** (created_at)
   - width: 140
   - 형식: "yyyy-MM-dd HH:mm"

9. **작업** (actions)
   - width: 120
   - 버튼:
     - ✏️ 수정 → `/benefits/{id}/edit`
     - 🗑️ 삭제 → confirm 다이얼로그 + 삭제

**그리드 설정**:
- 높이: 650px
- 페이지 크기: 10, 25, 50, 100 (기본: 25)

### 폼 필드 상세

#### 기본 정보 섹션 (Section 1)

1. **카테고리** (category_id) *필수
   - 컴포넌트: Select
   - 옵션: 혜택 카테고리 목록 (fetchBenefitCategories)
   - 표시: 카테고리 이름
   - 값: 카테고리 ID (UUID)

2. **제목** (title) *필수
   - 컴포넌트: TextField
   - fullWidth: true
   - 검증: 최소 1자

3. **부제목** (subtitle)
   - 컴포넌트: TextField
   - fullWidth: true
   - 선택사항

4. **기관명** (organization)
   - 컴포넌트: TextField
   - fullWidth: true
   - 선택사항

#### 신청 기간 섹션 (Section 2)

5. **신청 시작일** (application_period_start)
   - 컴포넌트: TextField
   - type: "date"
   - InputLabelProps: { shrink: true }
   - Grid: xs={12}, sm={6}

6. **신청 마감일** (application_period_end)
   - 컴포넌트: TextField
   - type: "date"
   - InputLabelProps: { shrink: true }
   - Grid: xs={12}, sm={6}

#### 공고 정보 섹션 (Section 3)

7. **공고일** (announcement_date)
   - 컴포넌트: TextField
   - type: "date"
   - Grid: xs={12}, sm={6}

8. **상태** (status) *필수
   - 컴포넌트: Select
   - 옵션:
     - draft: "임시저장"
     - upcoming: "예정"
     - recruiting: "모집중"
     - closed: "마감"
   - Grid: xs={12}, sm={6}

9. **주요 공고** (is_featured)
   - 컴포넌트: FormControlLabel + Switch
   - 기본값: false

#### 상세 정보 섹션 (Section 4)

10. **요약** (summary)
    - 컴포넌트: TextField
    - multiline: true
    - rows: 3
    - helperText: "최대 200자"

11. **썸네일 이미지** (thumbnail_url)
    - 컴포넌트: TextField
    - type: "file"
    - accept: "image/*"
    - 업로드: Supabase Storage (benefit-thumbnails/)
    - 파일명 형식: `benefit-thumbnails/{timestamp}-{random}.{ext}`

12. **외부 링크** (external_url)
    - 컴포넌트: TextField
    - type: "url"
    - placeholder: "https://example.com"

13. **태그** (tags)
    - 컴포넌트: TextField
    - helperText: "쉼표로 구분하여 입력"
    - 예: "청년, 주거, 서울"
    - 저장 형식: string[] 배열

#### 통계 섹션 (Section 5) - 수정 모드만

14. **조회수** (views_count)
    - 컴포넌트: TextField
    - type: "number"
    - 읽기 전용 (수정 모드에서만 표시)

### 버튼 영역

- **취소 버튼**
  - variant: "outlined"
  - onClick: navigate(-1) 뒤로가기

- **임시저장 버튼**
  - variant: "outlined"
  - onClick: status를 'draft'로 설정하고 저장
  - 표시: 수정 모드에서만

- **등록/수정 버튼**
  - variant: "contained"
  - type: "submit"
  - 텍스트:
    - 등록 모드: "등록"
    - 수정 모드: "수정"

### 폼 동작

**등록 모드**:
1. 빈 폼 표시
2. 제출 시 `createBenefitAnnouncement` 호출
3. 성공 시: Toast 메시지 + `/benefits/announcements` 이동
4. 실패 시: 오류 Toast 메시지

**수정 모드**:
1. `fetchBenefitAnnouncementById` 로 데이터 로드
2. 폼에 기존 데이터 채우기
3. 제출 시 `updateBenefitAnnouncement` 호출
4. 성공 시: Toast 메시지 + 목록으로 이동
5. 실패 시: 오류 Toast 메시지

**로딩 상태**:
- 데이터 로딩 중: CircularProgress 표시
- 제출 중: 버튼 비활성화 + 로딩 인디케이터

## 🎨 디자인 시스템

### 색상 체계
- **Primary**: Material-UI 기본 파란색
- **Success (Green)**: 활성 상태, 모집중
- **Error (Red)**: 비활성 상태, 마감
- **Info (Blue)**: 예정 상태
- **Default (Gray)**: 임시저장

### 상태 표현
| 상태 | 아이콘 | 색상 | Chip Color |
|------|--------|------|------------|
| 모집중 | 🟢 | Green | success |
| 마감 | 🔴 | Red | error |
| 임시저장 | ⚫ | Gray | default |
| 예정 | 🔵 | Blue | info |
| 활성 | 🟢 | Green | success |
| 비활성 | 🔴 | Red | error |

### 타이포그래피
- **H4**: 페이지 제목
- **Body2**: 일반 텍스트, 폼 레이블
- **Caption**: 부가 정보, 부제목

### 간격 (Spacing)
- Paper padding: 2 (16px)
- Section spacing: mb: 2 (16px)
- Grid spacing: 2 (16px)
- Button spacing: gap: 1 (8px)

### 컴포넌트
- **Button**: variant="contained" (primary), "outlined" (secondary)
- **TextField**: fullWidth, size="small"
- **DataGrid**: MUI X DataGrid
- **Chip**: size="small", fontWeight: 500
- **Paper**: elevation: 1 (기본)

## 📝 참고 사항

### 파일 업로드
- **Supabase Storage** 사용
- **버킷**: `benefit-thumbnails`
- **경로 형식**: `{folder}/{timestamp}-{random}.{ext}`
- **캐시 제어**: 3600초 (1시간)
- **중복 파일**: upsert: false (덮어쓰기 방지)

### 날짜 형식
- **표시 형식**:
  - 날짜: "yyyy-MM-dd" (예: 2024-01-15)
  - 날짜+시간: "yyyy-MM-dd HH:mm" (예: 2024-01-15 14:30)
- **로케일**: 한국어 (ko)

### 쿼리 키
- `['benefit-categories']`: 카테고리 목록
- `['benefit-category', id]`: 단일 카테고리
- `['benefit-announcements']`: 공고 목록
- `['benefit-announcement', id]`: 단일 공고

### 에러 처리
- **JWT 만료**: "세션이 만료되었습니다. 다시 로그인해주세요."
- **일반 오류**: Toast 메시지로 표시
- **로딩 상태**: CircularProgress 또는 버튼 비활성화
